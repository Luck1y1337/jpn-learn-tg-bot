# 日本語 Bot — Telegram-бот для изучения японского

Публичный бот для изучения японского языка по уровням JLPT (N5–N1):
изучение слов и кандзи, повторение по SRS, квизы, грамматика с примерами
и ежедневная рассылка нового материала. Интерфейс на трёх языках: RU / EN / UZ.

Тот же стек и стиль, что и у donation-бота «Поддержка Luck1y»:
aiogram 3.x, aiosqlite, деплой на Railway.

## Возможности

- **Онбординг** — выбор языка интерфейса, затем уровня JLPT.
- **SRS-повторение** (`/study`) — карточки «слово/кандзи → чтение и перевод»,
  оценка запоминания и пересчёт интервалов по упрощённому SM-2 (как в Anki).
- **Квизы** (`/quiz`) — короткие тесты на 4 варианта ответа по словам уровня.
- **Грамматика** (`/grammar`) — карточки конструкций со структурой и примерами.
- **Ежедневная рассылка** — в выбранное время бот присылает N новых слов,
  кандзи дня и грамматику дня; новые слова автоматически попадают в SRS.
- **Настройки** (`/settings`) — язык, уровень, время рассылки, число слов в день.
- **Прогресс** (`/stats`) — изучено, к повторению, стрик, точность квизов.
- **Донаты** (`/donate`) — поддержка через Telegram Stars.

## Источники контента

- **Кандзи** — [kanjiapi.dev](https://kanjiapi.dev) (без ключа).
- **Слова** — [jlpt-vocab-api](https://jlpt-vocab-api.vercel.app) (без ключа).
- **Грамматика** — статический набор в файле `grammar_points.json`
  (живого открытого API с грамматикой JLPT не существует).

Слова и кандзи тянутся «вживую», но результат **кэшируется в локальной SQLite**,
чтобы не грузить внешние API повторно и не зависеть от их аптайма.

### Переводы RU / UZ

Оба API отдают значения только на английском. Перевод на русский и узбекский
делается один раз при первом получении слова через библиотеку
`deep-translator` (Google-бэкенд, без ключа) и **кэшируется в базе навсегда**.
Для одного и того же слова переводчик повторно не вызывается.

## Структура проекта

```
bot.py             — точка входа, роутеры, запуск планировщика
config.py          — конфигурация из .env и константы
database.py        — SQLite: таблицы и запросы
util.py            — часовой пояс, работа с датой и временем
states.py          — FSM-состояния
locales.py         — тексты RU / EN / UZ
keyboards.py       — клавиатуры
common.py          — общие помощники (язык, уровень, is_admin)
middleware.py      — отсекает заблокированных пользователей
content_api.py     — запросы к kanjiapi.dev и jlpt-vocab-api + кэш
translate.py       — перевод EN → RU / UZ через deep-translator
srs.py             — алгоритм повторения (упрощённый SM-2)
scheduler.py       — ежедневная рассылка + ночной бэкап (APScheduler)
backup.py          — консистентный снимок базы, упаковка в zip, отправка админу
seed.py            — загрузка грамматики из JSON в базу
grammar_points.json — статический набор грамматики
handlers/          — обработчики команд (start, study, quiz, grammar,
                     settings, stats, donate, admin)
```

## Схема базы данных

- `users` — telegram_id, язык, уровень, время рассылки, число слов в день,
  дата последней рассылки, стрик, дата последнего занятия.
- `vocab_cache` / `kanji_cache` — кэш контента с внешних API и переводами.
- `grammar_points` — статический набор грамматики (3 языка + примеры).
- `srs_progress` — прогресс SRS: интервал, ease, повторения, дата показа.
- `quiz_results` — история квизов для статистики.
- `donations` — история донатов через Stars.

## Локальный запуск

1. Установите Python 3.11+.
2. Создайте и активируйте виртуальное окружение, установите зависимости:
   ```bash
   python -m venv venv
   venv\Scripts\activate        # Windows
   pip install -r requirements.txt
   ```
3. Скопируйте `.env.example` в `.env` и впишите `BOT_TOKEN` от @BotFather.
4. Запустите бота:
   ```bash
   python bot.py
   ```

На Windows можно просто запустить `run.bat` или `run.ps1`.

## Деплой на Railway

Сборка идёт по **Dockerfile** (задано в `railway.json`), поэтому Python 3.11
и `DB_PATH=/data/bot.db` фиксируются самим образом — ничего дополнительно
настраивать для этого не нужно.

1. **New Project → Deploy from GitHub repo** и выберите этот репозиторий.
   Railway подхватит `railway.json` и `Dockerfile` автоматически.
2. **Variables** — задайте переменные окружения:
   - `BOT_TOKEN` — токен от @BotFather (обязательно).
   - `ADMIN_ID` — ваш Telegram ID для `/admin` и `/broadcast` (можно несколько
     через запятую). Необязательно, но без него админ-команды выключены.
   - `TIMEZONE` — например `Asia/Tashkent` (по умолчанию `UTC`). Влияет на
     стрик и время ежедневной рассылки.
3. **Volume** — добавьте том и смонтируйте его в **`/data`**. Тогда база
   (`/data/bot.db`) и бэкапы переживут редеплой. Без тома данные сбросятся
   при каждом деплое.
4. Готово — Railway соберёт образ и запустит `python bot.py`. Логи видны
   во вкладке **Deployments**.

`BOT_TOKEN` — это секрет: задавайте его только в панели Railway, не коммитьте
в репозиторий (`.env` уже в `.gitignore`).

## Админ-панель (`/admin`)

Инлайн-панель для владельца бота (по образцу AI_bot), доступна пользователям
из `ADMIN_ID`. Разделы:

- **📊 Статистика** — пользователи всего и по уровням/языкам, заблокированные,
  карточки SRS, пройденные квизы, донаты, размеры кэша и число грамматики.
- **👥 Пользователи** — список с пагинацией и поиск по ID / username.
- **📢 Рассылка** — то же, что `/broadcast`: ввод → подтверждение → отправка.
- **🔨 Бан / разбан** — блокировка по ID; заблокированных отсекает middleware.
- **⚙️ Настройки** → **💻 Система** (CPU/RAM/диск/аптайм через psutil) и
  **🗄 Бэкап / восстановление** (см. раздел ниже).

Все действия проверяют, что вызвавший — админ; неадминов панель игнорирует.

## Бэкапы и восстановление

Бэкап — это **консистентный снимок базы** через SQLite online backup API
(`aiosqlite.backup()`), а не сырое копирование файла, поэтому его безопасно
делать прямо на работающем боте: «рваной» или повреждённой копии не будет.
Снимок упаковывается в zip и присылается админу в личку файлом.

- **Автоматически** — каждый день в **03:00** (часовой пояс `TIMEZONE`) бот
  присылает свежий бэкап админу (первому из `ADMIN_ID`).
- **Вручную** — команда `/backup` или кнопка **📦 Создать бэкап** в панели.
- **Восстановление** — команда `/restore` в ответ на сообщение с файлом
  (`jpn_backup.zip` или `.db`), либо кнопка **♻️ Восстановить**. Перед заменой
  бот проверяет, что файл — настоящая база этого бота (есть таблица `users`),
  и просит подтверждение: текущие данные **полностью заменяются**, это
  необратимо. Замена идёт тоже через online backup API.

> На Railway база и бэкапы лежат в томе `/data`; даже если том сбросится,
> последний бэкап всегда есть в личке у админа в Telegram.

## Заметки по SRS

Оценки при повторении: 🔴 «Не помню», 🟡 «Сложно», 🟢 «Легко».
Интервал следующего показа пересчитывается по упрощённому SM-2:
неверный ответ сбрасывает прогресс карточки, верные — растягивают интервал
с учётом коэффициента лёгкости (ease, минимум 1.3).

## Тесты

```bash
pip install -r requirements.txt
pytest
```

Тесты (`tests/`) покрывают слой базы (пользователи, SRS, квизы, кэш,
грамматика, бэкап/восстановление), алгоритм SM-2 (`srs.py`) и утилиты дат и
времени (`util.py`). Работают на временной БД в `tmp_path`, живую базу не
трогают.
