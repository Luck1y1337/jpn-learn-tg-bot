# 日本語 Bot — Telegram-бот для изучения японского

Публичный бот для изучения японского языка по уровням JLPT (N5–N1):
изучение слов и кандзи, повторение по SRS, квизы, грамматика с примерами
и ежедневная рассылка нового материала. Интерфейс на трёх языках: RU / EN / UZ.

Тот же стек и стиль, что и у donation-бота «Поддержка Luck1y»:
aiogram 3.x, aiosqlite, деплой на Railway.

## Возможности

- **Онбординг** — выбор языка интерфейса, затем уровня JLPT.
- **SRS-повторение** (`/study`) — карточки «слово/кандзи → чтение и перевод»,
  оценка запоминания и пересчёт интервалов по упрощённому SM-2 (как в Anki).
- **Квизы** (`/quiz`) — короткие тесты на 4 варианта ответа по словам уровня.
- **Грамматика** (`/grammar`) — карточки конструкций со структурой и примерами.
- **Ежедневная рассылка** — в выбранное время бот присылает N новых слов,
  кандзи дня и грамматику дня; новые слова автоматически попадают в SRS.
- **Настройки** (`/settings`) — язык, уровень, время рассылки, число слов в день.
- **Прогресс** (`/stats`) — изучено, к повторению, стрик, точность квизов.
- **Донаты** (`/donate`) — поддержка через Telegram Stars.

## Источники контента

- **Кандзи** — [kanjiapi.dev](https://kanjiapi.dev) (без ключа).
- **Слова** — [jlpt-vocab-api](https://jlpt-vocab-api.vercel.app) (без ключа).
- **Грамматика** — статический набор в файле `grammar_points.json`
  (живого открытого API с грамматикой JLPT не существует).

Слова и кандзи тянутся «вживую», но результат **кэшируется в локальной SQLite**,
чтобы не грузить внешние API повторно и не зависеть от их аптайма.

### Переводы RU / UZ

Оба API отдают значения только на английском. Перевод на русский и узбекский
делается один раз при первом получении слова через библиотеку
`deep-translator` (Google-бэкенд, без ключа) и **кэшируется в базе навсегда**.
Для одного и того же слова переводчик повторно не вызывается.

| Команда      | Что делает                             |
|--------------|----------------------------------------|
| `/start`     | Онбординг: язык → уровень              |

| `/grammar`   | Грамматика по текущему уровню          |
| `/settings`  | Язык / уровень / время / слов в день   |

## Структура проекта

```
bot.py             — точка входа, роутеры, запуск планировщика
config.py          — конфигурация из .env и константы
database.py        — SQLite: таблицы и запросы
util.py            — часовой пояс, работа с датой и временем
states.py          — FSM-состояния
locales.py         — тексты RU / EN / UZ
keyboards.py       — клавиатуры
common.py          — общие помощники (язык, уровень пользователя)
content_api.py     — запросы к kanjiapi.dev и jlpt-vocab-api + кэш
translate.py       — перевод EN → RU / UZ через deep-translator
srs.py             — алгоритм повторения (упрощённый SM-2)
scheduler.py       — ежедневная рассылка (APScheduler)
seed.py            — загрузка грамматики из JSON в базу
grammar_points.json — статический набор грамматики
handlers/          — обработчики команд (start, study, quiz, grammar,
                     settings, stats, donate)
```

## Схема базы данных

- `users` — telegram_id, язык, уровень, время рассылки, число слов в день,
  дата последней рассылки, стрик, дата последнего занятия.
- `vocab_cache` / `kanji_cache` — кэш контента с внешних API и переводами.
- `grammar_points` — статический набор грамматики (3 языка + примеры).
- `srs_progress` — прогресс SRS: интервал, ease, повторения, дата показа.
- `quiz_results` — история квизов для статистики.
- `donations` — история донатов через Stars.

## Локальный запуск

1. Установите Python 3.11+.
2. Создайте и активируйте виртуальное окружение, установите зависимости:
   ```bash
   python -m venv venv
   venv\Scripts\activate        # Windows
   pip install -r requirements.txt
   ```
3. Скопируйте `.env.example` в `.env` и впишите `BOT_TOKEN` от @BotFather.
4. Запустите бота:
   ```bash
   python bot.py
   ```

На Windows можно просто запустить `run.bat` или `run.ps1`.

## Деплой на Railway

1. Создайте новый проект из этого репозитория.
2. В переменных окружения задайте `BOT_TOKEN` (и при желании `TIMEZONE`).
3. Добавьте **Volume**, смонтированный в `/data`, — тогда база переживёт
   редеплой (в Dockerfile уже указан `DB_PATH=/data/bot.db`).

## Заметки по SRS

Оценки при повторении: 🔴 «Не помню», 🟡 «Сложно», 🟢 «Легко».
Интервал следующего показа пересчитывается по упрощённому SM-2:
неверный ответ сбрасывает прогресс карточки, верные — растягивают интервал
с учётом коэффициента лёгкости (ease, минимум 1.3).
